## 全案共通の設計原則

- **匿名(ペンネーム)運用**: note/Kindle/ココナラは表示名ペンネーム可・本人情報は運営側にのみ登録(検証済み)。ストアカは匿名制約が強いため全案で不採用
- **進み方**: 書いて売る(プラットフォームが集客) → 1対1で相談に乗る → 会員制の少人数の場、の順に育てる。ただし初月から人との関わりを組み込む
- **分業**: AI = 下書き・分析・記録・定型作業 / 本人 = 実践・独自データ・判断・関係構築
- **収益の現実線**: 初月0〜5千円、3ヶ月で月5千〜2万円、半年で月1〜3万円。これを下回っても「学習と発見」が得られていれば継続価値あり
- **法務**: 有料サービスの特商法表記は「請求があれば遅滞なく開示」方式で実質匿名運用が可能(体制は要整備・最終判断は専門家確認推奨)。金融商品の個別推奨は投資助言業に該当しうるため全案で行わない

## 案1: サブ4:30公開実験室 🏃(第一推奨)

**一言で**: 自分のマラソントレーニングを「AIで運営するプロジェクト」として公開運営し、同じ目標の停滞ランナーが集まる少人数の非同期部活に育てる。

| 項目 | 内容 |
|---|---|
| ターゲット | サブ4.5〜サブ4を目指して停滞している30-40代の時間がない市民ランナー(男性40-50代ランナーだけが増加中) |
| 提供価値 | 「才能でなく構造で走力を作る」過程の実況。Garminデータ×AI分析×PM手法(週次レビュー、リスク管理=故障予防)の実例 |
| 収益モデル | ①note有料記事(月2〜3本×300-500円) ②月額メンバーシップ(500-980円、noteメンバーシップ+Discord連携) ③様子を見てKindle化 |
| AIの仕事 | Garminデータの分析・可視化、週次レビューの下書き、記事ドラフト、部活の議事録・リマインド、練習メニューのたたき台 |
| 本人の仕事 | 走る(実践)、数字と体感の突き合わせ、メンバーの週次報告への構造化コメント、場のルール設計 |
| 差別化 | Garmin Connect+(月1,180円)がAI分析を提供済み→**分析そのものでは戦わない**。「同じ目標の仲間と自分の公開実験」という場に価値を置く |
| おもろさ | ◎ 自分が突き抜ける側に回る実験。速い人・ウルトラ勢との接点も生まれる。今年の目標(サブ4.5、新コミュニティ参加)と完全重複 |

**週4時間の業務分解**(練習時間は趣味なので事業時間に含めない)
- 土曜 90分: 今週のGarminデータをAIに分析させ、突き合わせて週次記事を仕上げ・公開
- 日曜 60分: 部活メンバーの週次報告に構造化コメント(AIが論点整理済みのものに判断を足す)
- 平日すきま 30分×2: X(ペンネーム)で記事の告知、メンバーの反応確認

**1ヶ月開始ロードマップ**
- 週1: ペンネームでnote/X開設。「サブ4:30公開実験室・キックオフ」記事(現状データ・目標・方法論)を無料公開
- 週2: 週次レビュー記事の型をAIと確立。有料記事1本目(例: 停滞ランナーの練習データの読み方)
- 週3: noteメンバーシップ開設(初期は月500円)。記事末尾で「同じ目標の人の非同期部活」を募集
- 週4: メンバー0〜3人でもDiscord部活を開始(自分の実験室として回る設計なので人数ゼロでも破綻しない)

**初期コスト**: note無料、Discord無料、AIサブスク(既存)。実質0円
**リスクと撤退基準**: 収益性は低い(月1万円到達に6ヶ月+)。3ヶ月でメンバー0人かつ自分が楽しくなければ型を変える(部活を畳んでコンテンツ専業へ)
**発見の実験として**: 「自分は観察者か、実践者か、場の主か」が3ヶ月で判明する

## 案2: 管理職の匿名壁打ち屋 📝(収益・ニーズ最強)

**一言で**: 「共感はしない。構造で返す」— 相談先のいない中間管理職向けの、非同期テキスト専門の壁打ちサービス。

| 項目 | 内容 |
|---|---|
| ターゲット | 相談先がない課長・係長クラス(94.9%がプレイングマネージャー、管理職希望者16.7%のみ。社内で弱みを見せられない層) |
| 提供価値 | 悩みを要件定義に変換して返す。感情の受容ではなく、論点の構造化・選択肢とトレードオフの提示。**相談側も匿名でいられる**のが価値 |
| 収益モデル | ①ココナラ非同期テキスト相談(3,000円/1テーマ・1週間、手数料22%) ②note実録記事(係長のリアル、300-500円) ③固定客がつけば月額顧問(3,000-5,000円/月) |
| AIの仕事 | 相談文の論点分解ドラフト、類似ケース・フレームワークの照合、回答の下書き、実録記事の下書き |
| 本人の仕事 | 当事者としての判断コメント、「それはやめた方がいい」と言う役、匿名化した実録の編集 |
| 差別化 | ココナラ相談カテゴリ4.3万件の大半は電話・共感型。**管理職特化×匿名×非同期テキスト×構造化**の組み合わせは空白(検証済み)。AIが肯定しかしない時代に「否定できる人間」 |
| おもろさ | ○ 他人の修羅場は突き抜けた話の宝庫。本業のマネジメント力に直接還流する |

**週4時間の業務分解**
- 土曜 2時間: 相談への回答作成(AI論点整理→本人判断、1件30-40分×2-3件)
- 日曜 1時間: 実録note記事1本(AIドラフト+編集)
- 平日すきま 1時間: 新着相談の受付確認・AIへの下調べ指示(即応は約束しない。「48時間以内返信」を明記)

**1ヶ月開始ロードマップ**
- 週1: ココナラ出品ページ作成(サービス設計・免責・返信SLA)。ペンネームnote開設
- 週2: 実録記事2本公開(信頼の種)。ココナラ出品開始
- 週3: 初回モニター価格(1,000円)で3件まで受付、レビューを貯める
- 週4: 相談対応+回答の型化(頻出パターンをAIに学習させる=メソッドAI化の種)

**初期コスト**: 0円(ココナラ・note無料)
**リスクと撤退基準**: 守秘・利益相反(自社社員からの相談が来たら辞退ルール)。重いメンタル相談は受けない旨を明記(カウンセリングではないと宣言)。3ヶ月で受注0件なら価格・出品文を再設計、6ヶ月で0件なら撤退
**発見の実験として**: 感情労働を排した対人支援が自分に持続可能かが判明する

## 案3: 「突き抜けた人」の構造分析ニュースレター 🔬(おもろさ最強)

**一言で**: ウルトラランナー、常識を捨てた生き方の人に非同期テキストで取材し、「なぜそれができるのか」をAIと構造分析して届ける研究誌。

| 項目 | 内容 |
|---|---|
| ターゲット | 平凡な日常の中で「極端な人の話」に飢えている読者(あなた自身がペルソナ)。取材対象は無名でも突き抜けた実践者 |
| 提供価値 | 単なる美談インタビューではなく「意思決定の構造・トレーニングの構造・生活の構造」の分析。取材される側には「自分の営みが言語化される」価値 |
| 収益モデル | theLetter(手数料17%)で無料+月額パート(480-980円)。当面は無料中心で読者を貯める設計 |
| AIの仕事 | 取材対象のリサーチ、質問設計のたたき台、回答の構造分析(意思決定モデル化)、記事ドラフト、取材依頼文 |
| 本人の仕事 | 取材相手の発掘と依頼(月1-2人)、質問の最終設計、対話、分析の妥当性判断 |
| 差別化 | 「極端な生き方特化×構造分析」の個人ニュースレターは競合不在(検証済み。ただし市場規模自体が未知数) |
| おもろさ | ◎◎ あなたのおもろさの源泉(突き抜けた人を見る)を事業の中核に置いた唯一の案。取材=1対1の構造化された対話そのもの |

**週4時間の業務分解**
- 土曜 2時間: 取材回答の分析・記事執筆(AIドラフト+編集)
- 日曜 1時間: 次の取材相手のリサーチ・依頼文送付(AIがリストアップ、本人が選定)
- 平日すきま 1時間: 取材の非同期やりとり(テキストで質問→回答)

**1ヶ月開始ロードマップ**
- 週1: theLetter開設。創刊準備号(コンセプト+第0号として「自分はなぜ突き抜けた人に惹かれるのか」)
- 週2: 取材候補10人リストアップ(X・ブログ・大会結果から)。依頼文送付(承諾率3割想定)
- 週3: 第1号取材の非同期インタビュー実施
- 週4: 第1号発行。SNSで取材相手にもシェアしてもらう(取材相手の読者が流入する構造)
- 補足: アスリートカンパニー練習会(今年の目標)が取材先開拓の場を兼ねる

**初期コスト**: 0円(theLetter無料開始可)
**リスクと撤退基準**: 収益化は最も遅い(半年は実質無収益を覚悟)。取材の承諾が取れないのが最大リスク→最初の2人は練習会等のリアル接点から。6ヶ月で購読100人未満なら位置づけを「趣味研究」に変更するか撤退
**発見の実験として**: 「知らない」と答えた2つの問い(価値観の核・憧れの正体)の答えが、取材を重ねるほど自分に返ってくる設計

## 案4: 出産・家庭運営PMキット 👶(旬・当事者性)

**一言で**: いままさに自分がやっている「出産準備のプロジェクトマネジメント」をテンプレート+実録として商品化する。

| 項目 | 内容 |
|---|---|
| ターゲット | 第一子を迎える共働き夫婦(特に「段取り担当」になった側)。夫婦の8割が「お金の話が負担」 |
| 提供価値 | WBS・リスク管理表・意思決定ログ・夫婦会議アジェンダといったPM道具を家庭に翻訳した実用キット+自分の実録 |
| 収益モデル | ①note有料記事(実録+テンプレ解説、500-980円) ②Kindle(『出産準備はプロジェクトである(仮)』、ペンネーム出版可・検証済み) ③テンプレ一式販売(note/BOOTH、1,000-2,000円) |
| AIの仕事 | テンプレの整備・汎用化、記事・書籍ドラフト、制度情報の下調べ(育休・手当) |
| 本人の仕事 | 実際に出産準備PMを回す(=生活そのもの)、実録の編集、購入者からの質問への非同期回答 |
| 差別化 | Notion出産テンプレは複数存在しレッドオーシャン気味(検証済み)→ **テンプレではなく「PM手法の家庭への翻訳」+実録**で差別化。Obsidian/Markdown版は空白 |
| おもろさ | ○ 家庭内PM(今年のホームラン目標)がそのまま事業になる。ただし突き抜け接点は薄い |

**週4時間の業務分解**
- 土曜 2時間: 実録記事またはKindle原稿(AIドラフト+編集)
- 日曜 1時間: テンプレの汎用化・改善(自家用→配布用)
- 平日すきま 1時間: 購入者質問への回答、制度情報のAI下調べレビュー

**1ヶ月開始ロードマップ**
- 週1: 自分の出産準備PMをObsidianで運用開始(既にやっていることの整理)。note開設
- 週2: 実録記事1本目無料公開(「出産準備をWBSにしてみた」)
- 週3: テンプレ第1弾(タスク一覧+リスク管理表)を有料記事化
- 週4: 反応を見てKindle企画を決定(売れ筋の切り口で)
- **注意**: 個別の金融商品推奨は書かない(投資助言業回避)。制度情報は一次ソース確認+「専門家確認を」の注記

**初期コスト**: 0円
**リスクと撤退基準**: 旬が短い(出産後は当事者性が「乳児育児」へ移る→続編化は可能)。競合テンプレとの価格競争。3ヶ月で販売10件未満なら実録コンテンツ専業に切替
**発見の実験として**: 「構造を与える人」としての自分の市場価値が最小コストで測れる

## 案5: 非エンジニアのObsidian導入伴走 🗂(実現性最堅)

**一言で**: 「Obsidianに挫折した非エンジニア」向けに、テンプレ販売と非同期の1対1セットアップ伴走を提供する。

| 項目 | 内容 |
|---|---|
| ターゲット | Notionは重い・情報が散らかる、と感じている士業・コンサル・研究職・ライター等の非エンジニア(Obsidian商用無償化で拡大期) |
| 提供価値 | 用途特化のVaultテンプレ(読書ノート/案件管理/日次レビュー)+2週間の非同期伴走(質問し放題ではなく週2回のまとめ回答) |
| 収益モデル | ①ココナラ「Obsidianセットアップ伴走」(5,000-8,000円/2週間) ②テンプレ+ガイド(note/BOOTH、980-1,980円) ③導入記事(note) |
| AIの仕事 | テンプレ・ガイドの整備、質問回答のドラフト、顧客のつまずきパターン集約 |
| 本人の仕事 | Vault設計の判断、顧客の用途ヒアリング(非同期フォーム)、週2回の構造化回答 |
| 差別化 | ココナラ先行は1名(4,000円/60分の同期ビデオ型・検証済み)→ **非同期伴走型・用途特化テンプレ**でずらす。AI連携(Claude連携メモ術)は本人の得意技 |
| おもろさ | △〜○ 得意技ど真ん中で勝ちやすいが、突き抜け接点は薄い。知的な楽しさはある |

**週4時間の業務分解**
- 土曜 2時間: 伴走顧客への週次まとめ回答(2-3人分)、テンプレ改善
- 日曜 1時間: 導入記事・ガイド執筆(AIドラフト+編集)
- 平日すきま 1時間: 新規問い合わせ対応、AIへのつまずきパターン集約指示

**1ヶ月開始ロードマップ**
- 週1: 自分のVault構成を汎用化した「読書ノート特化テンプレv1」作成
- 週2: note導入記事2本無料公開(検索流入の種)。BOOTHでテンプレ販売開始
- 週3: ココナラで伴走サービス出品(モニター価格3,000円×2名)
- 週4: モニターのつまずきデータでテンプレとガイドを改訂、正価に引き上げ

**初期コスト**: 0円
**リスクと撤退基準**: 市場規模が読めない(Obsidian日本語人口のデータ薄)。ツールのアップデートへの追従コスト。3ヶ月で販売+受注が計5件未満なら、テーマを「AI×知識管理」に広げてピボット
**発見の実験として**: 「教える・伝える」型の関わりが自分に合うかが測れる

## 社長への提言(COOとして)

1. **週4時間で回せるのは1案です。**5案は選択肢であってポートフォリオではない。まず1つに絞ることを強く推奨します
2. **私の第一推奨は案1(サブ4:30公開実験室)**。理由: おもろさ(自分が突き抜ける実験)・今年の個人目標との完全重複・練習時間が事業時間を兼ねる構造的な効率・失敗しても実践記録が資産として残る
3. **収益を早く見たいなら案2(壁打ち屋)**。統計的ニーズ最強・空白確認済み・ココナラで初月受注の現実味が最も高い
4. 案1と案3は将来合流できる(実験室の発信が取材の信用になる)。案2と案4も合流できる(構造を与える人ブランド)。**最初の1案は「入口」であって「終点」ではない**
5. 選定の判断基準: 採点表の7列は制約適合として私が保証します。残る1列「おもろさ」— **どの案なら週末の朝、眠くてもやりたいか** — だけが社長にしか判断できない問いです
