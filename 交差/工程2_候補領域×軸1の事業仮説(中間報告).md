# 交差工程2: 候補領域×軸1の事業仮説生成(中間報告)

作成日: 2026-07-09。作業指示書 §3「次の作業」のステップ2(候補領域×軸1)とステップ3のうち統括AIが行える部分(8要件のうち「おもろい」を除く7要件の評価)にあたる。サブエージェントは使わず統括AIが直接作成し、判断過程を全て開示する。

**前提**: 工程1で確定した候補14領域(社長承認済み・v4)に、軸1「1人AI企業の7類型」を掛けて「領域×型」の事業仮説を生成した。**「おもろい」の評価欄は空欄である。指示書ルール8のとおり、おもろいかどうかは社長にしか判定できないため、統括AIは採点していない。**

---

## 1. 方法と、型レベルの事前チェック(判断の開示)

軸1の7類型(「何を売るか」の分類)を各候補領域に掛ける前に、類型そのものが社長の制約(週末2〜4時間・月〜1万円・匿名・1ヶ月以内開始)と両立するかを、軸1調査の**事実**に基づき確認した。

| 型 | 名称 | 制約との適合(統括AIの判断と根拠) |
|---|---|---|
| 型1 | プロダクト型(SaaS・アプリ等) | **限定的に可**。軸1の事実: 検証可能な高収益が集中するが「全員実名・フルコミット寄り」。本格SaaSは週4時間の保守・障害対応と両立しない。**テンプレート・小型ツールの販売に限定**すれば可(この形は実質、型2の販売形態に近い) |
| 型2 | コンテンツ・作品販売型 | **最も適合**。軸1の事実: 参入最速・最安(数日で開始可)・匿名可・非同期完結。収益は極小〜中だが、目標(月数千〜数万円)とは整合 |
| 型3 | メディア・広告型 | **除外**。軸1の構造的発見: 検証可能な成功例は例外なくチーム化(6〜22人)しており、広告収益は規模が前提。「AIを使って1人のまま」の検証可能例は補充調査でも未発見。発信自体は他の型の集客手段として使うが、広告収益モデルは採らない |
| 型4 | 受託・代行型 | **限定的に可**。軸1の事実: 収益の確実性は高いが納期拘束が構造的に発生 → 「即応・常時接続を避ける」制約と衝突。**非同期・週末納品を明示した小口案件に限定**すれば可 |
| 型5 | 専門知識サービス型 | **可**。軸1の事実: 「自分の専門性・メソッドをAIに載せて提供量を拡張する」共通進化パターン。関わり方(1対1の構造化された対話)と最も親和。弱点は匿名での信頼構築 → 実践データの開示で補う設計 |
| 型6 | 実店舗・物理サービス型 | **除外**。物理制約(施術・製造・発送・立地)が週4時間・月1万円と両立しない |
| 型7 | 配信・パーソナリティ型 | **限定的に可**。軸1の事実: 匿名×検証可能な収益が最も成立している型。ただしライブ配信は即応・常時接続に近づくため、**非同期のキャラクター発信・少人数コミュニティ運営に限定** |

→ 以降、各領域には主に**型2・型5(+限定付きの型1・型4・型7)**を掛ける。型3・型6を掛けなかった理由は上表のとおり。

**評価記号の定義**(毎回この意味で使う): ◎=強く適合 / ○=適合 / △=条件付き・懸念あり / ×=不適合。評価対象の7要件は: 要件1=週4時間以内 / 要件2=1ヶ月以内にスモールスタート / 要件3=AIフル活用 / 要件4=社長のやることが人との関わりで性質を活かせる / 要件5=関心領域 / 要件7=ニーズ増大 / 要件8=ニッチ。**要件6=おもろい、は社長の判定欄として空けてある。**

## 2. 領域ごとの交差と仮説(全23仮説)

各領域について「どの型が成立し、どの型が成立しないか」を先に判断し、成立する組み合わせだけ仮説にした。

### 候補1: ビジネスケアラーの介護段取り

- 型の適合: 型2○(段取り知識は教材化できる)/ 型5○(1対1の構造化対話に最適)/ 型4×(介護は緊急性があり受託すると即応要求が発生)
- **仮説1**〔候補1×型2〕**「親の介護、はじめの90日」段取りキット** — 介護の発生から体制構築までを、要件定義・タスク分解・関係者マップ・お金の一覧表で構造化したガイド+テンプレート(note/Kindle+Notion/Excelテンプレ)。AIの役割: 制度・手続きの網羅調査、ケース別の分岐整理、改訂。本人の役割: PM視点での型づくりと読者からの質問への非同期回答
- **仮説2**〔候補1×型5〕**介護段取りの非同期壁打ち** — 状況をテキストで聞き取り、段取り表と次の一手を返す1対1サービス。感情のケアには踏み込まず「プロジェクトとしての介護」に限定する旨を明示して募集する。AIの役割: ヒアリング整理・制度調査・段取り案の下書き。本人: 最終判断と質問設計

### 候補2: 実家じまい・相続・空き家の段取り

- 型の適合: 型2○ / 型5○ / 型4×(士業独占業務との線引きが受託形態では曖昧になる)
- **仮説3**〔候補2×型2〕**「実家じまいプロジェクト」段取りキット** — 相続発生前後〜家の処分までの工程表・関係者調整・費用一覧・専門家(士業)への依頼の仕方を構造化。登記・税務の代行はせず「全体の段取りと専門家への橋渡し」に限定
- **仮説4**〔候補2×型5〕**実家じまいの非同期壁打ち** — 仮説2と同型。個別の法律・税務判断はせず段取りに限定

### 候補3: フリーランス・副業の制度対応と基盤づくり

- 型の適合: 型2○(実践記録そのものが商品になる)/ 型5△(自分がまだ初心者のため提供できる専門性が薄い)
- **仮説5**〔候補3×型2〕**「匿名・週4時間の副業実験」公開ログ** — 本事業そのものの意思決定・収支・制度対応(開業届・確定申告・インボイス)・AI活用を、実名を伏せて記録・販売する実践記録。AIの役割: 記録の整理・分析・執筆支援。**注: これは単独案であると同時に、他のどの案を選んでも併走できる「メタ案」である**(統括AIの補足)

### 候補4: 家計の生活防衛・家計運営のPM化

- 型の適合: 型2◎(実データが最大の差別化)/ 型5○ / 型1△(家計簿テンプレは型2の販売形態として内包)
- **仮説6**〔候補4×型2〕**「家計を複式簿記で回す」教材+テンプレート** — 実際に運用している複式簿記家計・月次決算・資産ポートフォリオ設計の型を、実データ(匿名化)つきで教材化。個別の投資助言(金融商品取引法の登録業務)はしない
- **仮説7**〔候補4×型5〕**家計の月次決算 非同期レビュー** — 相手の家計を構造化し「家計の月次決算」を一緒に立ち上げる1対1。助言ではなく仕組みづくりに限定

### 候補5: 生成AI時代のリスキリング・AI活用実戦知

- 型の適合: 型2△(激戦区。一般向けAI教材は差別化不能 → 切り口を絞る条件付き)/ 型4△(非同期小口に限定)
- **仮説8**〔候補5×型2〕**「PM・中間管理職のためのAI実務活用」教材** — 一般向けではなく、要件定義・リスク管理・チーム運営という自分の実務文脈に絞ったAI活用の型。60超サービス比較の実戦知を下敷きに
- **仮説9**〔候補5×型4〕**小規模事業者向けAI業務フロー構築の小口代行** — 非同期・週末納品を明示した定型案件のみ。軸1の事実(ココナラAI出品で2ヶ月0円→月5,000円)が示すとおり立ち上がりは遅い

### 候補6: カスハラ対応・職場マネジメントの仕組みづくり

- 型の適合: 型2○(2026年10月義務化で時限性の強い実務需要)/ 型5△(組織向けは実名信頼が必要になりがち、個人管理職向けに限定)
- **仮説10**〔候補6×型2〕**カスハラ対応の現場マニュアル雛形+導入ガイド** — 義務化(経過措置なし)に間に合わせたい小規模事業者・現場責任者向けの、対応フロー・掲示文・記録様式の雛形集。AIの役割: 指針・他社事例の網羅調査と雛形起こし
- **仮説11**〔候補6×型2/5〕**「共感に頼らないマネジメント」実務教材** — 組織行動論×中間管理職の当事者経験(労務管理・心理的安全性施策)を、感情労働ではなく仕組みで回すマネジメントの型として教材化+管理職の非同期1対1壁打ち

### 候補7: AI詐欺・偽情報からの防衛

- 型の適合: 型2○ / 型7△(注意喚起キャラクター発信はあり得るが単体では収益形が弱い)
- **仮説12**〔候補7×型2〕**「親のスマホを詐欺から守る」設定・会話キット** — 現役世代が帰省時に親のスマホを防御設定し、詐欺の手口を家族で共有するための手順書+会話のきっかけ集。AIの役割: 最新手口の継続調査と改訂(手口の変化が速い領域なので更新性が価値になる)

### 候補8: 育児・共働き家庭運営のPM化

- 型の適合: 型2◎(進行中のライフイベントがそのまま実践データになる)
- **仮説13**〔候補8×型2〕**「出産準備プロジェクト」実践記録+テンプレート** — 現在進行中の出産準備(ベビー用品リサーチ・資金計画・タスク)をPMの型で記録し、リアルタイムの実践記+テンプレとして販売。AIの役割: リサーチ・比較表・記録整理
- **仮説14**〔候補8×型2〕**共働き家庭運営のPM化教材** — 家事・行事・保活・家計をプロジェクト管理する型(家庭内PM)。個人目標「家庭内PM」との一致

### 候補9: ひとり単位の生活設計(単身現役層)

- 型の適合: 型2△(手法は移植できるが当事者でない弱点が商品の説得力に直結)
- **仮説15**〔候補9×型2〕**ひとり暮らしの生活設計テンプレート** — 家計・防災・健康・もしもの備えを1人分に最適化した設計キット。**統括AIの注記: 当事者性の欠如が工程1から一貫した弱点**

### 候補10: つながりすぎない自己コントロール

- 型の適合: 型2○ / 型5△
- **仮説16**〔候補10×型2〕**「つながらない時間」の仕組みデザイン教材** — 通知・端末・習慣の設計をGTD/習慣化の枠組みで構造化。意志力ではなく仕組みで解決する型。本人の「即応を避ける生き方」の実践がデータになる

### 候補11: 持久系スポーツの趣味縁・少人数コミュニティ

- 型の適合: 型7/場◎(関わり方③「共通目的のある少人数の場」の本命)/ 型2○
- **仮説17**〔候補11×型7/場〕**データで走る市民ランナーの非同期クラブ** — サブ4等の共通目標を持つ少人数(5〜15人)が、Garmin等のデータと週次ふりかえり(YWT)を非同期で共有する会員制の場。本人は場の設計者かつ一実践者として自分のサブ4:30挑戦を晒して走る。AIの役割: データ整理・ふりかえりの構造化・練習メニューの下調べ(医学的指導はしない)。**軸2の「場を設計する人+同じ道を走る人」の在りたい姿と最も一致する仮説**(統括AIの判断)
- **仮説18**〔候補11×型2〕**市民アスリートのトレーニングデータ分析記** — 自分の実データ×AI分析の記録を発信・教材化。仮説17の入口・集客としても機能

### 候補12: AI×個人の学び方・知識管理

- 型の適合: 型2◎ / 型1△(テンプレ販売として型2に内包)/ 型5○
- **仮説19**〔候補12×型2〕**Obsidian×AIの知的生産システム教材+テンプレート** — ツェッテルカステン×Claude等の実運用(自分のVaultの設計)を教材・テンプレとして販売。海外に先行者はいるが日本語×AI統合はまだ薄い
- **仮説20**〔候補12×型5〕**知識管理の1対1設計セッション** — 相手の学び・仕事のフローを聞き取り、その人の知的生産システムを一緒に設計する構造化された非同期対話

### 候補13: 住まい・家庭の災害リスク管理

- 型の適合: 型2◎(GIS素養を匿名で活かせる数少ない形)/ 型5△(不動産助言との線引きを厳格に)
- **仮説21**〔候補13×型2〕**「住まいの災害リスク読み解き」ガイド** — ハザードマップ・地形・水災保険料率の読み方を、住まい選び・住み替え判断の手順に落とした教材。個別物件の推奨はしない(不動産・保険業法の外側に留まる)
- **仮説22**〔候補13×型2〕**家庭防災のPM化キット** — 備蓄・避難・連絡・資金をプロジェクト化し年次見直しまで設計する家庭向けテンプレ。乳児のいる家庭版など当事者性のある派生も可能

### 候補14: 現役世代の健康習慣づくり(運動・睡眠)

- 型の適合: 型2○ / 型5○(伴走は「励まし」でなく「仕組みの調整」として設計すれば感情労働を避けられる)
- **仮説23**〔候補14×型2→型5〕**データ駆動の運動・睡眠習慣化プログラム** — 自分のGarmin実データで語る「ゼロから運動習慣」の型+非同期の仕組み調整伴走。医学的指導・治療はしない。仮説17(既に走る人の場)と対をなす「これから始める人」向け
