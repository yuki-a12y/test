# 交差工程3: マーケティング調査結果 — 横断比較と戦略判定(中間報告)

作成日: 2026-07-10。工程3計画(社長承認済み)に基づき、5案それぞれの需要・競合をサブエージェント5体(sonnet)で並列調査した。調査原文・投入プロンプト・全検索クエリ(計151本)は `交差/工程3_調査原文/` に全量記録済み。本文書は統括AIによる監査・横断比較・判定であり、**事実(調査結果)と統括AIの判断・意見を区別して書く**。

## 1. 統括AIによる調査の監査(網羅性・偏りチェック)

- **クエリ監査**: 5エージェント計151クエリを確認。全エージェントが需要側・競合側・無料代替・販路・規制を横断しており、大きな抜けは検出されなかった。「売れる」という結論を先に埋め込んだクエリもない(競合の商品名検索は競合特定のための正当な使用)
- **反証の収集**: 5報告すべてが反証(売れない兆候)を明示的に含んでおり、指示どおりの規律で調査されている
- **系統的な限界(重要)**: note・ココナラ・BASE等の個別商品ページは全エージェントでBot対策(HTTP 403)により直接閲覧できず、**「個別商品が実際にいくらで何件売れているか」のデータが5案すべてで系統的に欠落**している。つまり「ニッチが空いている」という判定は「検索で直接競合を発見できなかった」という消極的証拠に依存しており、これ以上は机上調査では詰められない。→ **統括AIの判断: 追加調査ではなく、各案とも「小さく出して実測する」工程をロードマップに組み込むべき**(1ヶ月スモールスタートの要件2とも整合する)
- **検証性の規律**: 「Kindleで月5〜7万円」等の自己申告収益は全報告で除外・低ラベル化されており、軸1調査の教訓(合成事例汚染)は守られている

## 3. 案別の判定と戦略(統括AIの判断 — 根拠は§2と調査原文)

判定基準は社長が示した「**ニッチをとりに行けるかどうか**」+8要件との整合。

### 案1 複式簿記家計教材 → **成立。ただし切り口を1段絞る**
- 戦略: 「複式簿記の家計簿テンプレ」(先行あり)ではなく、**「家計の月次決算を回し続ける運用の型」+「簿記×資産ポートフォリオの統合」**(いずれも商品化の空白を確認済み)を正面に立てる。実データ(匿名化)の開示が最大の差別化
- 入り方: note実用系(売れ筋価格帯1,800円)+テンプレ。Kindleは第2弾
- 留意: 個別の投資助言(金融商品取引法)に踏み込まない設計は工程2から維持

### 案2 働きまくりの人の健康習慣 → **成立。5案中で需要とニッチの均衡が最良**(統括AIの評価)
- 戦略: 「多忙な会社員×ウェアラブル実データ×仕組み調整(励ましなし)」の三点掛け合わせを維持。教材は「プログラム」ではなく**「実データ公開つきの実践記録+仕組みの型」**として売る(継続の壁への構え: 続けさせることを約束しない、仕組みの設計図を売る)
- 入り方: note連載(無料で実データ記事→有料で型・テンプレ)
- **必須の設計条件**: 「治す・改善・予防」等の医学的効能表現を使わない(薬機法・医療広告ガイドライン・健康増進法)。ココナラは医療隣接規約+手数料30%のため使わない

### 案3 家庭防災PMキット → **単独では「ニッチをとりに行けない」と判定。縮小して条件付き存続を提案**
- 根拠: 同コンセプトの確立競合(書籍+YouTube18万人)、官製無料の直接カバー、類似テンプレの値付け天井0〜300円。工程2スコアは高かったが、市場側の裏付けが5案中最も弱い
- 縮小案: **「乳児のいる家庭の防災立ち上げ」に特化**(唯一確認された隙間)し、案4の方向(a)と同じ読者に売る姉妹商品として設計(販売基盤・発信を共有し週4時間内に収める)
- 代替の選択肢: 案3を案4に吸収して5案目を空け、工程2の生き残り仮説から繰り上げる(ただし残る△は「難易度高そう」等の懸念つきのみ)

### 案4 共働き家庭運営のPM化 → **方向(a) 新生児期の家庭立ち上げ、と判定**
- 根拠(3方向比較の事実から): (b)保活は行政デジタル化+専業サービスが先行し、待機児童8年連続減で需要自体が縮小トレンド。(c)家事分担は特許取得済み無料アプリ・官製無料ツール・既存テンプレ販売まで揃い3方向で最厚。(a)は先行が実質1社(世帯経営ノート、Amazonレビュー7件=可視実績小)で、PM型テンプレの直接競合は未確認
- 出生数減という母数の反証はあるが、年67万件の新規発生が毎年続く「必ず初心者が生まれ続ける」市場であり、ニッチ規模としては十分(統括AIの判断)
- 当事者性(現在進行中の出産準備・実データ)が最大の差別化。工程2で提示した3方向のうち、データと当事者性の両方が揃うのは(a)だけ

### 案5 管理職の実践記録 → **「とりに行けるが、5案中最もリスクが高い」と判定。検証ファースト設計を条件に存続を提案**
- 空いているもの: 「匿名現役管理職×共感に頼らない×失敗込みの実録」の三点掛け合わせ(直接競合未発見)。無料代替は「型」の情報であり、実録はそれと重ならない
- リスク: 課金実証が実質1件・識学との混同・身バレと当事者性のジレンマ
- 条件(統括AIの設計提案): (1)識学と混同されない立ち位置を明示 — 「感情の排除」ではなく「**共感に頼らずに心理的安全性をつくる仕組み**」(排除ではなく設計) (2)最初の3ヶ月は無料連載で反応を実測し、数値基準(§5)を満たしたら有料化 (3)実録の匿名加工は「時期・業界のぼかし」に留め、事実の合成はしない

## 4. 5案共通の重要事項(社長に必ず伝えるべき発見)

1. **「匿名」は表示上のみ成立し、法的には半匿名**: 特定商取引法上、有料販売を反復継続する個人は「販売業者」に該当しうる。プラットフォーム設定で住所・電話番号の非表示は可能だが、**氏名は開示請求への対応義務が残る**(複数の二次情報が一致。note公式ページの直接確認は403で未完 — 検証可能性: 中)。**副業禁止グレーという前提に直結するため、開始前に確認が必要**(統括AIの意見: noteの最新の特商法表記の仕様確認と、勤務先の副業規定の再確認を1ヶ月ロードマップの最初の週に置くべき)
2. **note収益の現実**: 収益化経験者は登録者938万人中約20万人、収益ありの年間中央値1,200〜2,000円(検証可能性: 中)。目標(月数千〜数万円)は「上位でなくても届く」水準だが、ゼロが普通の世界であることは前提にすべき — 「収益化を急がない」方針と整合
3. **実売データの系統的欠落**: 「ニッチが空いている」は全案とも消極的証拠。だからこそ全案に**撤退基準つきのテスト販売**(小さく出して実測)を組み込む
4. 販路の基本形は全案共通: **note(手数料10〜20%)を主戦場、Kindleを補助**。ココナラ(手数料30%・対人前提)は使わない

## 5. 社長への確認事項(判断材料と判断基準つき)

### 確認1: 案4の方向を「(a)新生児期の家庭立ち上げ」で確定してよいか
- 判断材料: §2・§3および調査原文(案4)の3方向比較表
- 判断基準: 「ニッチをとりに行けるか」— (b)(c)は競合・行政の厚みで否定的データが揃った。(a)は先行1社・可視実績小

### 確認2: 案3(家庭防災)の扱い — 3択
- (i) **「乳児家庭の防災立ち上げ」に縮小して存続**(案4(a)と読者・販売基盤を共有する姉妹商品)← 統括AIの推奨(意見)
- (ii) 案4(a)に吸収して5案目を空け、工程2の生き残りから繰り上げ
- (iii) 現構想のまま提案書化(撤退基準を厳格化)
- 判断基準: 調査が示した競合の厚さ・値付け天井(0〜300円)をどう見るか

### 確認3: 案5の「検証ファースト」条件つき存続でよいか
- 判断材料: §3案5の項(空き=形式、リスク=課金実証1件・識学混同・身バレ)
- 判断基準: リスクを取ってでも「おもろい」か。社長の△コメント「語れるレベルではないので、よりニッチなら」に対する回答が「教材でなく実録+3ヶ月無料テスト」という設計になっているか

### 確認4: 匿名性の限界(特商法の氏名開示義務の可能性)を了解のうえ進めるか
- 判断材料: §4-1。表示上のペンネームは全販路で可能だが、法的な完全匿名は保証されない
- 判断基準: 副業規定との関係で許容できるリスクか(必要なら開始前に専門家確認を挟む選択肢もある)
