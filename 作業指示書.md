# 作業指示書: 新規事業5案プロジェクト(新スレッド引き継ぎ用)

作成日: 2026-07-09。前スレッドのコンテキストが重くなったため、この指示書を起点に別スレッドで継続する。**新スレッドの統括AIは、作業開始前にこの文書と `資料/` 配下の3文書を必ず読むこと。**

---

## 1. プロジェクトの目的

依頼者(以下「社長」)の新規事業案を5個提案する。事業の要件は8つ:

1. 社長1人で週4時間以内で回せること
2. 1ヶ月以内に開始できるスモールスタートであること
3. AIをフル活用すること
4. 社長がやるのは人との関わりで、社長の性質を活かせること
5. 社長の関心のある領域であること
6. 社長が「おもろい」と感じること
7. これからニーズが増大する領域であること
8. ニッチであること

## 2. 方法論(社長が確定させたフレーム)

ビジネス案は、**独立に調査した3軸の交差点**として導く:

- **軸1: 1人AI企業**(世の中にどんな形が存在するか) → 調査完了。`資料/軸1_1人AI企業の類型.md`
- **軸2: 社長の人柄**(ヒアリング) → 完了・本人確認済み。`資料/軸2_社長プロファイル.md`
- **軸3: 社会のニーズ**(市場調査) → 調査完了。`資料/軸3_社会のニーズ領域マップ.md`

各軸は**他の軸を混ぜずに純粋に調査する**(交差を収集段階に前倒ししない)。3軸とも調査済みで、**現在地は「交差工程の直前」**。

## 3. 現在地と次の作業

### 完了済み
- ヒアリング4ラウンド → 軸2確定
- 軸1調査(事例38件・7類型に帰納。検索クエリ185本を開示させ統括AIが網羅性チェック・補充済み)
- 軸3調査(2段階: 13の実在分類体系→7観点を合成・本人承認→観点別sonnet×7体で99ニーズ領域+横断12クラスタ)
- 旧提案書(プロセス欠陥により却下)は `archive/proposals/` に保存

### 次の作業 = 交差工程(提案済み・**本人未承認** — 着手前に承認を取ること)
1. **軸3×軸2**: 99領域をプロファイル(関心・制約・関わり方4形態・利益相反・消耗回避)でフィルタし候補領域を1〜2割に絞る。絞った結果と**絞った理由を全件開示**して中間報告
2. **候補領域×軸1**: 残った領域に7類型を掛け、「領域×型」の事業仮説を生成
3. **評価**: 8要件でスコアリング。ただし**「おもろい」だけは統括AIは採点せず、社長の直感で判定してもらう**
4. 上位を本人確認 → 5案に絞って具体化(収益モデル・AI/本人分業・週4hの業務分解・1ヶ月ロードマップ・リスクと撤退基準) → 提案書をコミット
- 交差は調査ではなく判断の工程なので、サブエージェントを使わず統括AIが直接行い、判断根拠をすべて開示する

## 4. 働き方のルール(前スレッドで社長の指摘により確立。違反するとやり直しになる)

1. **答えを先に決めない**: 仮説の言葉(事業形態名・候補案・プラットフォーム名)をサブエージェントの検索指示に入れない。調査は要件・条件だけで行い、結論は結果から帰納する。「答えで検索して答えを作る」循環は禁止
2. **観点の調査と具体の調査を分ける**: 網羅を主張する調査は、まず「観点(分類体系)」自体を実在するものから調査・合成して本人承認を取り、その観点で具体を調べる
3. **抽象度を設計する**: 具体(個別ニーズ=数万個)で網羅は不可能。網羅可能な中間抽象度(ニーズ領域など)を明示的に設定する
4. **網羅性チェックは統括AIの仕事**: サブエージェントの自己申告に任せない・鵜呑みにしない。結果が世の中の常識的な多様性より狭くないかを自分で判定し、抜けは追加調査を指示する。サブエージェントには使用クエリ一覧を報告させ、偏りを検証する
5. **サブエージェントのプロンプトは投入前に本人承認を得る**。モデル選定は理由付きで提示する(定型収集はsonnet、品質担保は統括側で行う設計が基本)
6. **中間報告・相談を各節目で行う**: ヒアリングだけでなく、リサーチ結果・分析結果も都度共有して方向性を合わせる。ただし判断を求めるときは**「判断材料」と「判断基準」を必ず添える**(勘で答えさせない)。統括AIが自分で判断すべきことを本人に丸投げしない
7. **勝手な記号・略語を使わない**(P1、型Aなど)。使う場合は毎回定義する。社長の質問には正面から答える — 質問を無視して先に進むと却下される
8. **「おもろい」の判定は社長にしかできない**。統括AIの推し・意見と、事実・調査結果を明確に区別して提示する
9. **成果物はMarkdownでリポジトリにコミット**(ブランチ: `claude/solo-business-proposals-5z0j7a`)。Web版は作らない

## 5. 社長の制約(要旨 — 詳細は軸2資料)

- 副業禁止・グレーの会社員 → **匿名(ペンネーム)前提**、収益化を急がない
- 週末にまとめて2〜4時間 / 月〜1万円 / 発信基盤ゼロ / 日本国内メイン
- 本業(自治体向けSIer・GIS)との利益相反を避ける
- 消耗する事柄(避ける): 意味を感じない作業、感情労働・共感の要求、即応・常時接続
- 収益目標は月数千〜数万円で十分。目的は自己成長+独立の種まき+おもろさ

## 7. 注意事項

- 資料内の出典はスペース節約のため「出典名+中核数値」に圧縮している。数値を提案書等で引用する際に一次確認が必要なら再検証すること
- 軸3の網羅性チェックで「予防医療・健康づくり」「ペット関連」「葬送・墓じまい」が薄い可能性を未検証ラベル付きで記載済み。交差で必要になれば追加調査
